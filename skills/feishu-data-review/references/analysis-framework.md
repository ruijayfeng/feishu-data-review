# 数据复盘分析框架

通用数据分析方法论，适用于任何飞书表格的数据复盘场景。

## 列类型识别规则

| 检测标准 | 类型 |
|---------|------|
| 80%+ 值可解析为日期 → `time` | 时间列 |
| 80%+ 值可解析为数字 → `numeric` | 数值列 |
| 唯一值 < 50% 且有数据且唯一值 ≤ 30 → `categorical` | 分类列 |
| 其他 → `text` | 文本列 |

## 数值列统计指标

- 总量、均值、中位数、标准差、最大最小值、25/75 分位数、IQR、偏度
- 趋势：首尾 1/3 均值变化率（方向 + 百分比）
- 异常值：超出 1.5 倍标准差的数据点

## 分类对比

- 按分类列分组聚合每个数值列的 `count / sum / mean / min / max`
- 显著差异检测：组间均值极差超过总体均值 20% 时标记为显著差异
- 仅分析唯一值 ≤ 30 的列作为分类列

## 相关性分析

- 数值列之间计算 Pearson 相关系数
- ≥ 15 对停止
- ≥ 0.7 标记为显著相关性

## 洞察提取优先级

| 优先级 | 类型 | 触发条件 |
|-------|------|---------|
| 1 | 异常值 | ≥ 5 个数据点超出 1.5 倍标准差 |
| 1 | 说明 | 文本列唯一值 > 90%，说明排除原因 |
| 3 | 强相关性 | Pearson |r| ≥ 0.7 |
| 3 | 趋势 | 趋势方向 |
| 4 | 显著分类对比 | 分类列差异 > 20% |
| 4 | 趋势幅度 | 变化 > 10% |

## 报告撰写规范

**结构模板：**

```markdown
# [数据源名称] 数据复盘报告

> 分析时间：YYYY-MM-DD | 数据范围：[行数] | 数据来源：[链接]

## 一、数据概览
[一段话总结数据整体情况]

## 二、关键发现
[3-5 个发现，每个包含：现象描述 + 数据支撑 + 解读]

## 三、趋势分析
[如果有时间列，描述趋势变化]

## 四、对比分析
[如果有分类列，描述类别差异]

## 五、相关性发现
[如果有强相关性，说明联动关系]

## 六、建议与下一步
[2-3 条可执行建议]
```

**写作原则：**
1. 每个观点必须有具体数据支撑，禁止空话
2. 突出对比和变化（"比上月增长"、"是平均值的 2 倍"）
3. 洞察按重要性排序，最重要的放前面
4. 建议必须可执行，避免"建议加强"这类废话
